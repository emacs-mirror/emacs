Solution Idea

obsolete

RB , 1995-08-15

Purpose

An "idea" document records an idea for a solution which has some sort of impact on the project requirements. An idea might be a fragment of design, a method for doing something, a new data structure, or whatever.

Content

The content of an idea document is just a text document. The text should include some sort of justification or analysis of the idea with respect to requirements.

Tagging

Ideas are tagged "idea.<name>" where <name> is a mnemonic name for the idea.
