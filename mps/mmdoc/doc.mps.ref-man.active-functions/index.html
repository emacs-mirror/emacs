List of Functions on Stack When a Call to MPS is in Progress

incomplete

lmb, 1997-10-13

Manipulation of a running program, such as a debugging, requires knowledge of the state of the program. The MPS has complex state, especially when hardware barriers are used, and is not re-entrant. Debuggers might wish to know when a program is "inside MPS". It is therefore necessary to supply a list of functions that are used to enter the MPS, and one of which is found on the stack whenever a call to the MPS is in progress. This list of functions is as follows:
