                      RULES FOR USER DOCUMENTATION
                                rule.doc
                               draft rule
                           gavinm 1997-02-25

Scope: This document gives rules which any user documentation must comply with.


.use: It must provide enough information to enable an appropriately competent 
user to use any feature.

.find: It must be easy to find any piece of information in the documentation.

.expect: It must not mislead the user about what behaviour to expect.

.aware: It must make the user aware of appropriate features.

.market: If appropriate, it must be suitable for use in marketing; that is, it 
must readily present features in a good light.

.cons: It must be consistent with actual behaviour.

.min: It must contain no superfluous material.

.complete: It must contain all appropriate material.
