                         THE DEFAULT C COMPILER
                               builder.cc
                             draft builder
                           richard 1995-08-16

Pre-processor symbol: MPS_BUILD_CC


This build environment is the default C compiler on Unix platforms.  In other 
words, the version of "cc" which is shipped by default with the platform OS.  
For example, under Irix 4 and 5 this is SGI's MIPS C compiler.

Notes, that under SunOS or Solaris this is the "bundled" C compiler, which is 
not usually very useful, and not usually supplied with Solaris anyway (see 
builder.sc).

Here's a recommended way to invoke SGI's Irix cc compiler:
cc -c -ansiposix -D_BSD_TYPES -fullwarn -g
(this comes from ScriptWorks, vaguely)
I'm not sure we need -ansiposix (-ansi ought to do us) and -D_BSD_TYPES

Note that 'cc' on Irix 4 is quite different from that on Irix 5, and in fact it 
is not reliable. Here you can see it mis-parsing an expanded assertion:

cc -DTARGET_ARENA_CLIENT -ansi -DTARGET_VAR_DF -g -c -o i4r4cc/df/mpsisw.o 
mpsisw.c
cc -DTARGET_ARENA_CLIENT -ansi -DTARGET_VAR_DP -g -c -o i4r4cc/dp/mpsisw.o 
mpsisw.c
accom: Error: , line 1: Incomplete external data declaration (missing storage 
class) or function definition missing () before {function body}
       rcIdStruct.file, 812);             } while(0)  ;  
       ----------^
accom: Error: , line 1: syntax error
       rcIdStruct.file, 812);             } while(0)  ;  
       ----------^
cc -DTARGET_ARENA_CLIENT -ansi -DTARGET_VAR_DS -g -c -o i4r4cc/ds/mpsisw.o 
mpsisw.c
cc -DTARGET_ARENA_CLIENT -ansi -DTARGET_VAR_RO -O -c -o i4r4cc/ro/mpsisw.o 
mpsisw.c 

So on Irix 4 I recommend using GCC instead [NickB, 1997-02-20]
