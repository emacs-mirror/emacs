Rules: rule.universal

Tag template: status.<code>


Tagging:

Status descriptions must be tagged by "status." followed by the status code 
they describe.  For example, "status.obsolete" describes what it means for a 
document to be obsolete.  All status codes must be adjectives.


Purpose:

A document of type status describes the meaning of a document status code.  For 
example, status.draft describes what is means for a document to be of "draft" 
status.
