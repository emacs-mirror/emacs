INTRODUCTION

.intro: This document describes various conventions for code in the MPS.

.purpose: The document is intended to be used for development, approval, and 
review.

.readership: This document is intended for anyone writing or reading MPS source.


CONVENTIONS

.range: Where it is necessary to store a range (especially of addresses or 
indexes), it should be stored as the semi-open interval [base, limit).  
.range.drawback: Note that this has the drawback that it is impossible to 
represent a range of addresses that includes the highest address.
