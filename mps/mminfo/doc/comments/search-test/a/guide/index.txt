.intro: Use comments.search-test.* documents to test the search facilities of 
the substrate of the InfoSys (currently Lotus Notes).  The tests are designed 
to show how word-variants (ie. word-stemming), quoting, grouping, 
capitalisation, ordering, punctuation, repetition, etc, are treated.

.docs.other:
a.guide         This guide document.  The "a" is so it is obvious and lists 
first alphabetically.
a.latest        Notes indexes are not updated instantly.  After editing any of 
.docs.test, 
                you should change the "a.latest" document to contain a new 
random word.  
                Use the "Full Text" pane of "File/Database/Properties..." to 
request an 
                update to the MMInfoSys index.  Then you can search on the 
random word 
                to tell when the indexes have definitely been updated.  
Warning: 
                this might be defeated if re-indexing is not done in order.
