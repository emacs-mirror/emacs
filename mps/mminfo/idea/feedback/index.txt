                        MEASUREMENT AND FEEDBACK
                             idea.feedback
                               draft idea
                           richard 1995-07-28

SUMMARY

Provide a method for the client programmer to observe the memory
manager in order to make better use of it and thereby meet the
client's particular requirements.  Feedback includes diagnostics about
misuse, measurements, visualization of structure.


DESCRIPTION


( - Feedback Model Diagram )


Typical requirements are to optimise:
 - CPU time
 - storage
 - code volume

Other issues:
 - Measureables
 - Controllables
 - Measurement in DylanWorks environment
 - Deslugging


JUSTIFICATION

- Meta-requirement: adaptability to unknown requirements
