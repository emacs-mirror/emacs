                  PROCEDURE FOR COMMENCING CHECK-IN OF
                 ASSIGNMENT TO "TRUNK" IN MM EVOLUTION
                     proc.assignment.begin-checkin
                               draft proc
                           gavinm 1997-03-13


.intro: See proc.darwin for scope, readership, update procedure, and additional 
procedural notes.  See also guide.assignment.

.check.rel: Check that the change has not been rescheduled (see 
proc.change.reschedule ) to a different release from that in which the 
assignment was started.

.open: Open the change document and enter edit mode.

.status: Set the status of the assignment to "Releasing".

.report: Fill in a record of the changes (see proc.assignment.end.record). 

.files: Append a report of the files (or whatever) changed to "Files Changed".

.close: Close and save the change document.  Give a change comment such as 
"Begun release of assignment N".
