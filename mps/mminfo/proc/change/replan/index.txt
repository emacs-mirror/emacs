         PROCEDURE TO REPLAN A REJECTED CHANGE IN MM EVOLUTION
                           proc.change.replan
                               draft proc
                           gavinm 1997-06-11


.intro: See proc.darwin for scope, readership, update procedure, and additional 
procedural notes.  See also guide.change.

.open.change: Open the change document and enter edit mode.

.open.request: Open the request document and enter edit mode.

.replan: Choose a child release (or sibling releases) to rework the change in.

.create: Execute proc.change.create for each of the chosen releases.

.close.request: Close the request document.  [I don't think saving is 
necessary.  GavinM 1997-06-11]

.status: Set the status of the replanned change to "Rescheduled" if any changes 
were created, or "Abandoned" otherwise.

.close.change: Save and close the original change document with a comment like 
"Rescheduled for release.XXX" or "Abandoned because ...".
