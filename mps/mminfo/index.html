Memory Pool System Project

Memory Management Information Database

Nick Levine, Ravenbrook Limited, 2001-10-08

1. Introduction

This document catalogues the Memory Management Information Database.

The readership of this document is anyone interested in the project.

This document is Ravenbrook company confidential.

2. The Database

xml.zip Zipped xml-like document containing text of all the other documents (in other words, everything but the attachments).
attachments/ Flat directory of files attached to one or more documents in the Notes database.
The following directories each correspond to sets of documents with common forms.
arch/ Architectures.
bib/ Bibliographies.
builder/ Build systems.
check/ Checklists.
design/ Design documents.
doc/ Internal documentation of the MM group. This is a large class - everything which was not specifically given a different form has ended up here.
importance/ Importance.
issue/ Issues. These predate Darwin (the product evolution database).
mail/ Mail archives of the MM group.
obj/ Objectives of the MM group.
os/ Operating systems.
person/ People.
platform/ Platforms.
proc/ Procedures.
review/ Project reviews.
role/ Roles within the MM group.
rule/ Rules of the MM group.
stage/ Stages.
symbol/ Documentation of symbols (all of which currently under mps/c).
type/ Project types.
urgency/ Urgencies.
