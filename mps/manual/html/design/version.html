18. Software versions¶

18.1. Introduction¶

.intro: This is the design of the support in the MPS for describing and inspecting versions.

18.2. History¶

.hist.0: Incomplete document. David Jones, 1998-08-19.

.hist.1: Converted from MMInfo database design document. Richard Brooksby, 2002-06-07.

.hist.2: Converted to reStructuredText. Gareth Rees, 2013-03-11.

18.3. Overview¶

.overview: There are three different sorts of version under consideration:

  1. versions of the (MPS) library used (linked with);
  2. versions of the interface used (header files in C) when compiling the client’s program; and
  3. versions of the documentation used when the client was writing the program.

There are issues of programmatic and human access to these versions.

.overview.split: The design is split accordingly. See design.mps.version-library for the design of a system for determining the version of the library one is using. And other non-existent documents for the others.
